# 导出模板系统

## 概述

导出中心 (`export.html`) 提供两种导出模式：
1. **Flat 导出**：传统行列式 Excel，用户自定义字段模板
2. **日历月报导出**：按部门+员工列排布的日历格式月报

自 v1.0.21 起，XLSX 生成不再使用浏览器端 SheetJS，改为由 Python `export_server.py`（基于 openpyxl）在服务端生成带样式的 XLSX。前端从 IndexedDB 获取数据后以 JSON 形式 POST 到 Python API，再下载返回的 XLSX blob。

### API 端点

| 方法 | 路径 | 用途 |
|------|------|------|
| POST | `/api/export/flat` | 平铺报表导出 |
| POST | `/api/export/calendar` | 日历月报导出 |

## 单元格样式规则

`export_server.py` 中的 `_get_cell_style` 函数对单元格值进行正则匹配，自动应用样式：

| 正则匹配 | 样式 | 说明 |
|----------|------|------|
| `请假\|出差\|加班\|补卡` | 蓝色字体 (FF0066CC) | OA 标注类文本 |
| `迟\|早` | 红色字体 (FFFF0000) | 迟到/早退标记 |
| `^\d{1,2}:\d{2}` | 红色字体 (FFFF0000) | 时间值（迟到/早退打卡） |

此外，休息日（status=rest 或节假日非工作日）所在行整行填充灰色底色 (FFD9D9D9)，在 `build_calendar_report` 中直接对整行单元格设置 `cell.fill = GRAY_FILL`，不受 `_get_cell_style` 返回值影响。

## Flat 导出

### 模板设计

用户可在导出中心编辑模板字段：
- 添加/删除列
- 自定义列名
- 从 17 种可用字段中选择数据源
- 支持 `_index`（自动序号列）
- "另存为模板" 保存到 `export_templates` 表

### 可用字段

| 字段名 | 说明 |
|--------|------|
| _index | 自动序号 |
| employeeNo | 考勤号码 |
| name | 姓名 |
| department | 部门 |
| date | 日期 |
| period | 对应时段 |
| scheduleStart | 上班时间 |
| scheduleEnd | 下班时间 |
| signIn | 签到时间 |
| signOut | 签退时间 |
| lateMinutes | 迟到时间 |
| earlyMinutes | 早退时间 |
| overtimeHours | 加班时间 |
| travelHours | 出差时间 |
| absent | 是否旷工 |
| status | 状态 |
| leaveType | 请假类型 |

### 导出流程

1. 用户选择月份（或全部）→ 从 `attendance_results` 查询
2. 按模板字段映射数据
3. 前端将 records + template JSON POST 到 `/api/export/flat`
4. Python `build_flat_report` 用 openpyxl 生成 XLSX 并应用单元格样式
5. 前端接收 XLSX blob 并触发浏览器下载

## 日历月报导出

XLSX 由 Python `build_calendar_report`（openpyxl）生成。前端将 `targetMonth` + `fields` + `results` + `schedules` + `holidays` 以 JSON POST 到 `/api/export/calendar`，接收 XLSX blob 后触发下载。

### 格式说明

按企业月考勤月报模板设计：
- **行结构**：每天 2 行（上午行 + 下午行），整月共 2×N 行
- **列结构**：日期列 + 排班列 + 打卡时间列 + 各部门员工列
- **前 3 列合并**：日期、排班、打卡时间在 2 行中合并为一格

### 表头结构

```
Row 1: 日期 | 排班 | 打卡时间 | 市场营销部 | (空) | 技术部 | ...
Row 2: (空) | (空) | (空)     | 梁XX      | 覃XX | 张三   | ...
```

部门列标题跨员工合并（`merge_cells`）。

### 单元格内容规则

| 条件 | 上午行 | 下午行 |
|------|--------|--------|
| 有打卡签到 | signIn 时间 | signOut 时间 |
| 有 OA 标注 (请假/加班/出差/补卡) | 标注文本 / 拼接 | 下午有签退则填写签退 |
| 休息日 (status=rest) | 空 | 空 |
| 请假 (status=leave) | "请假"/"请假/类型/Nh" | 空 |
| 出差 (status=travel) | "出差"/"出差/Nh" | 空 |
| 缺勤 (absent=true) | "缺勤" | 空 |
| 加班 (status=overtime/suspect_ot) | "加班"/"加班/Nh" | 同有签退逻辑 |

迟到/早退分钟数拼接到签到/签退时间后，格式为 "08:30 迟5min" / "17:00 早3min"，触发红色字体样式。

### 休息日样式

休息日所在行（上午行 + 下午行）的所有单元格统一填充灰色底色 (FFD9D9D9)，由 `build_calendar_report` 在生成时对 `is_rest==True` 的日期行直接设置 `cell.fill = GRAY_FILL`。

### 日期列

日期列（A 列）显示中文格式如 "6月1日"，不再使用 Excel 序列号计算。

### 排班判定

月报中的排班列通过 schedules 表和 holidays 数据共同判定：

- 优先检查节假日：若 `holiday_map` 中存在该日期，使用 `holiday.name` 覆盖排班标签，`holiday.isWorkday` 决定是否为工作日
- 其次检查 schedules：存在 `workDays[day] === true` 的记录 → "工作日"；不存在 → "休息日"
