# 考勤规则引擎

## 概述

规则引擎 (`shared/rules.js`) 是系统的核心计算模块，负责将打卡数据、排班表、OA 申请数据综合处理，为每位员工每天产出考勤结果。

## 6 种状态体系

| 状态 | ID | 含义 | 判定条件 |
|------|-----|------|----------|
| normal | 正常 | 正常出勤 | 有打卡记录 且 无迟到/早退/旷工 (或被容错豁免) |
| rest | 休息 | 休息日 | 排班表标记为休息 (isWorkDay = false) |
| abnormal | 迟到 | 异常迟到 | 签到时间晚于排班上班时间+阈值 |
| leave | 请假 | 请假 | 有对应日期的 leave_records 且 leaveDays >= 1 或 无打卡记录 |
| travel | 出差 | 出差 | 有对应日期的 travel_records |
| absent | 缺勤 | 旷工 | isWorkDay 且 无打卡 且 无请假/出差/漏打卡 |

## isRestDay 与 status 的关系

这是两个易混淆的概念，在 v1.0.10 中已明确区分：

| 字段 | 含义 | 由什么决定 | OA 事件是否影响 |
|------|------|-----------|----------------|
| `isRestDay` | 排班列显示"休息"还是"上班" | **仅由排班表 + 假期管理** | 否 |
| `status` | 当天考勤结果状态 | 排班 + OA 数据 (漏打卡/请假/出差/缺勤) | 是 |

**示例**：某员工排班为休息日，但因漏打卡提交了说明——此时 `isRestDay = true`（排班休息），`status = normal`（漏打卡解释后状态正常）。排班列正确显示"休息"（而非被覆盖为"上班"），追溯弹窗可查看关联漏打卡记录。

## 迟到/早退判定

```js
// _calcDeviation(signIn, signOut, config)
if (signIn > workStartTime + lateThreshold) → lateMinutes = signIn - workStartTime
if (signOut < workEndTime - earlyThreshold) → earlyMinutes = workEndTime - signOut
```

阈值在设置页配置。每次打卡记录独立计算偏差，按天汇总。

## 容错豁免规则

每月迟到满足以下两个条件时，该员工所有迟到记录清零，状态恢复为 normal：

1. 迟到次数 ≤ `graceTimes` (默认 2 次)
2. 累计迟到分钟 ≤ `graceMinutes` (默认 30 分钟)

豁免在 `calculateMonth()` 末尾对整月汇总后一次性执行。

## 加班计算

每人每天加班 = 打卡记录中的 overtimeHours 之和 + 关联加班 OA 记录的 overtimeHours：

```js
// 打卡自带加班
totalOvertime + p.overtimeHours

// OA 加班记录追加
if (overtimeRecord) adjustedOvertime += overtimeRecord.overtimeHours
```

## 加班结余 (Carry Over)

每月计算一次结余：

```text
prevBalance = 上月结余 (查 carry_over 表, prevMonthKey 为 targetMonth 的上个月)
adjustmentHours = 本月调休请假消耗的小时数 (leaveType 含"调休"的 leaveHours 之和)
monthOvertime = 本月打卡自带加班 + OA 加班记录
newBalance = prevBalance + monthOvertime - adjustmentHours
```

结余不能为负数（`Math.max(0, newBalance)`）。

**prevMonthKey 查找**：向前查 2 个月（`month - 2`），因为当月 carry_over 在月计算完成后才会写入，不能引用自身。

## isWorkDay 判定

```js
_isWorkDay(schedulesData, holidaysData, dateStr)
```

优先级（从高到低）：
1. 假期管理中有该日期 → `holiday.isWorkday ? true : (holiday.isHoliday ? false : ...)`
2. 排班表 `workDays[day]` → `true` (上班) / `false` (休息)
3. 无排班数据 → 默认上班 (`return true`)

## 缺勤 (absent) 判定

仅在同时满足以下全部条件时判定为缺勤：

- `isWorkDay === true` (上班日)
- `dayPunches.length === 0` (无任何打卡记录)
- 无请假记录
- 无出差记录
- 无漏打卡说明

## 漏打卡处理

有漏打卡 OA 记录 (miss_punch_records) 时，`status` 设为 `normal`（不影响 `isRestDay`）。

漏打卡支持"漏打卡已补"场景：匹配 logic 为申请人姓名（`m.applicant === employeeName`）或忘打卡人员姓名（`m.missPerson === employeeName`）与 `missDate` 匹配。

## 调休请假扣减

当 `leaveRecord.leaveType` 包含"调休"关键字时：
- `adjustedOvertime = adjustedOvertime - leaveRecord.leaveHours`（从当日加班中扣除）
- 同时在 `_updateCarryOver()` 中从月结余扣除

## 配置变更检测 (v1.0.10)

考勤页面加载时自动检测：
- `settings` 表中 `config_updated_at` 时间戳
- 对比 `last_calc_{YYYY-MM}` 时间戳
- 若配置变更时间晚于最后计算时间 → 显示黄色横幅提示重新计算
