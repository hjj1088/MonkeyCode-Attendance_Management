# 数据文件类型与格式

## 概述

系统支持 6 种 Excel 文件类型导入。文件类型通过 `Excel.identifyFileType()` 根据表头关键词**自动识别**，无需用户手动选择。

## 类型识别规则

识别流程：
1. 提取第一行表头，去空格
2. 按优先级依次匹配规则
3. 必含表头全部匹配 → 有效类型
4. 加分表头越多 → 得分越高 → 优先级越高
5. 无匹配 → `unknown` 类型，显示表头供用户确认

### 1. 打卡记录 (punch)

**必含**：`考勤号码`、`签到时间`

**加分**：`签退时间`、`迟到时间`、`部门`、`日期`、`上班时间`、`下班时间`

**标准化字段**：
```
employeeNo, name, date, period, scheduleStart, scheduleEnd,
signIn, signOut, lateMinutes, earlyMinutes, absent,
overtimeHours, workHours, department
```

### 2. 请假记录 (leave)

**必含**：`请假类型`、`开始日期`

**加分**：`结束日期`、`请假天数`、`申请人`、`申请部门`

**标准化字段**：
```
applicant, department, leaveType, startDate, endDate,
leaveDays, leaveHours, reason
```

**特殊处理**：结束日期为空时按开始日期处理（单日请假）。

### 3. 加班记录 (overtime)

**必含**：`加班起止时间`

**加分**：`申请人`、`申请部门`、`加班内容`

**标准化字段**：
```
applicant, department, startTime, endTime, overtimeHours, content
```

**特殊处理**：通过 `overtimeHours` 字段获取小时数，匹配到考勤日时追加到当日加班。

### 4. 出差记录 (travel)

**必含**：`出差起止日期`

**加分**：`申请人`、`目的地`、`出差事由`、`出差人员`

**标准化字段**：
```
applicant, department, destination, travelers,
startDate, endDate, travelType, reason
```

**特殊处理**：日期字段可能为"2026-02-01至2026-02-03"格式，需按 `-`、`~`、`至`、`到` 分割。

### 5. 漏打卡记录 (miss_punch)

**必含**：`忘打卡日期`

**加分**：`申请人`、`忘打卡人员`、`未打卡时间`、`事由`

**标准化字段**：
```
applicant, department, missDate, missPerson, missTime, cardTime, reason
```

### 6. 排班表 (schedule)

排班文件不通过表头识别，而是通过 **Sheet 名称匹配**：Sheet 名符合 `^\d{1,2}月$` 格式（如 `1月`、`12月`）即为排班表。

**特殊处理**：
- 通过单元格**填充色**判断休息日：有填充色 = 休息，无填充色 = 上班
- 排班区间按 `(周一~周日, 周一~周日)` 格式解析
- 入库时**按员工展开**：每条排班 × 员工数量 → 每人每月一条记录

## 数字日期/时间格式

Excel 中日期/时间可能以数字形式存储：
- **日期数字** (如 `44971`) → 通过 `XLSX.SSF.parse_date_code()` 转为 `YYYY-MM-DD`
- **时间小数** (如 `0.729166667`) → 转为 `HH:MM`（乘 24 得小时，小数部分乘 60 得分钟）

## 去重策略

导入时检查 `raw_files` 表，同名文件已存在则跳过。同一类型文件导入时先清空对应表再写入（打卡数据除外，打卡导入会触发 `Matcher.syncEmployees()` 重建员工表）。
